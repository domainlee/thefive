Installation (HTML)

Follow the steps below to setup your site template:

  1. Unzip the downloaded package and open the HTML folder to find all of the HTML template files. You will need to upload these files to your hosting web server using FTP or localhost in order to use it on your website.
  2. Below is the folder structure and needs to be uploaded to your website or localhost root directory:
    • HTML/css - Stylesheet files.
      • HTML/css/bootstrap.css - Bootstrap v4.5.0 framework.
      • HTML/css/main.css - Main stylesheet files.
      • HTML/css/custom.css - Add your custom stylesheet in this file.
    • HTML/image - Images files.
    • HTML/fonts - ALL font external files.
    • HTML/js - Core javacript files.
      • HTML/js/custom.js - Activation JS file.
      • HTML/js/vendor.min.js - Vendor JS file.
    • HTML/lang - Language system files.
    • HTML/mailer - Ajax contact form files.
    • HTML/plugins – All external libs.
    • HTML/template-part – Main navbar, Footer navbar and header language files.
    • HTML/index.html - Homepage

  3. You should upload all or specific HTML files as per your need.
  4. You are good to go for editing the template & adding your content now!

HTML Language Translation System

We used in EMYUI a language translation system. We used I18next a JavaScript Internationalisation framework
to translate or add new language to your html pages, read this.

All pre-made translations file in HTML/lang folder inside a .JSON file, which can be edited by any text editor.

Translations

For ex: English Language
Open the HTML/lang/en/lang.json file with any text editor and edit the values according to your needs. And here is a small example of how JSON looks 

		  
{
"systems": {
  "langname": "EN"
},

"genaral": {
  "homapgetitle": "Create a unique, powerful & professional website",
  "homepage": "Homepage",
  "soon": "soon",
  "getstarted": "Get started now",
  "getstartedfree": "Get started free",
  "pages": "Pages"
}

you only need to add a parent name and then put translations texts inside.
The prefix keys homapgetitle, homepage, soon .. this example file are individual words separated by hyphens and written in lowercase ONLY.
to use these translations add an HTML tag on the the element that you want to put the text on it with data-i18n attribute.

For Example:


<span data-i18n="[html]genaral.homepage"></span>
<span data-i18n="[html]genaral.getstarted"></span>
<span data-i18n="[html]genaral.homapgetitle"></span>

Add new language

  1. CREATE new folder for your language, for example ar in HTML/lang folder and add inside it lang.json file.
  2. ADD your translations inside lang.json like we mentioned earlier.
  3. ADD you new lang link inside HTML/template-part/languages.html file, there you can find and change the translation buttons. 
    
<a href="#" data-lng="ar">Arabic</a>

how to install WHMCS

Before installing the template, you need to install WHMCS first.

WHMCS theme installation

Before installing the template, please make sure you unpacked the archive and extracted template files.

  • documentation: this folder has the docs files
  • HTML: this folder has the all EMYUI HTML template files
  • WHMCS (X.X.X): this folder has EMYUI the all template & orderforms files
  • changelog & updates.txt: this txt file has all updates and changelogs of the template

Plese make sure that template version, which you are trying to install is compatible with your WHMCS installation!

Let’s walk you through how to install the Coowhm WHMCS template properly. To complete this tutorial, you will need your FTP login credentials for your hosting environment, your favorite FTP client, and finally the download files from themeforest. I would recommend either FileZilla for Windows/Mac

Let’s head over to the local folder in which you saved the theme download. The file is a ZIP archive and will need to be extracted. Mac users can easily do this by double-clicking on the .zip file, and it will unzip the contents into a folder with the same name as the download. Windows users can use the default ZIP extraction tool, or you can use something like WinZip or WinRar to do this for you. Either way, once you’re done, you will have a folder with the theme contents. Fire up your favorite FTP client and connect to your hosting environment. You will need to navigate to the root directory of your WHMCS installation. If you’re not sure what the root directory is, it will be the folder that contains the config.php file for your WHMCS installation.

In the FTP client, you can locally browse to the unzipped folder of the template download. Inside of the download files in WHMCS (X.X.X) folder, you will find another folder called “upload.” This folder contains all of the files that need to be uploaded to the root WHMCS installation directory. However, do not upload the “upload” folder, just the files inside of it! Once the upload is complete, we can go through the process of installing your template and orderform.


Changing your default template & orderform

When you’re ready for your clients to see your new template, you will need to head to “Setup” > “General Settings” and wait for the page to load. Look for the option called “Template” and change it to “EMYUI theme” Save your changes, and the new template will be live!.

Our WHMCS template support all WHMCS order forms & Our own orderform also: EMYUI Cart

You will need to change your default order form template to our if you want to use it. To do this, you will head to “Setup” > “General Settings” > “Ordering.” Once the page loads, you will select the order form and click “Save Changes.”

Common WHMCS issues

you fill find in this section common WHMCS issues and some suggestions to fix them

If you see an Oops error page like this indicates an error occurred during the generation of the page.
The error can be anything, from a simple notice or warning to a fatal error that halts execution.
Or you are installing the wrong theme version.
It's the PHP error reporting level on your server that determines which types of errors will trigger it.

If you find some error installing the theme like Missing main.css, white page or something similar, please:

  • make sure you are uploading the correct file WHMCS(X.X.X)/upload.zip.
  • make sure that you unpack the zip file after uploading to your server
  • make sure that you are using the right version for your WHMCS
  • make sure that you activate the template from your WHMCS admin panel
  • make sure that the files has the right permission

Or probably has these issues in your server

  • Syntax errors in custom modules, hooks or templates.
  • Incompatible hooks or addons.
  • PHP, Apache or Ioncube related errors.
  • Missing or corrupted files / incomplete uploads.
  • If the above suggestions do not solve please contact your hosting provider, or our support team will help you with your issue

Embedded Video
